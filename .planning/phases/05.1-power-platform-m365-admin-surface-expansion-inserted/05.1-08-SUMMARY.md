## Decisions Made

1. **Three-shape product detection** (prefix + workload + preset) — gives operators flexibility to express product-scoped mutations in whatever form matches their workflow. All three shapes collapse to the same discriminator.

2. **`'mixed'` sentinel over JSON array** — keeps the Postgres JSONB query `meta->>'product' = 'mixed'` trivial. Alternative (JSONB array) would require `?` containment operators and complicate grep-based retention.

3. **`PRODUCT_POLICIES` contains only strict entries** — absence = permissive. Keeps the policy map as a declaration of exceptions (exo + sp-admin) rather than duplicating the default thresholds for each permissive product.

4. **`classifyByAlias` uses hardcoded prefix literals, NOT `PRODUCT_AUDIENCES` iteration** — `coverage-check.mjs` is a Node ESM .mjs module loaded by bin/generate-graph-client.mjs; importing the TypeScript `PRODUCT_AUDIENCES` would require a compile step and complicate the codegen wiring. The 5 prefix literals are stable; CR review catches any drift.

5. **GAP-POWER-PLATFORM.md committed via `git add -f`** — parallels GAP-EXCHANGE-ADMIN.md / GAP-GRAPH-API.md (`.planning/` is gitignored but research artifacts are CR-reviewed inputs to the coverage harness).

6. **Integration test at `src/lib/admin/__tests__/` rather than `test/admin/`** — matches the existing convention (4 sibling enabled-tools-*.int.test.ts files). `test/admin/` does not exist in the repo. See Deviations below.

4. **inferProductFromSelectors** computes the product discriminator from the selector list.
5. **withTransaction** runs SELECT FOR UPDATE + UPDATE + writeAudit atomically — `meta.product` lands in `audit_log.meta` JSONB.
6. **Redis publishToolSelectionInvalidation** fires post-commit (not exercised in Tests A1-A7 due to MemoryRedisFacade being side-effect-only, but the no-throw path is verified).
7. **pgPool read-back** shapes the updated tenant row via `tenantRowToWire` and returns 200.
