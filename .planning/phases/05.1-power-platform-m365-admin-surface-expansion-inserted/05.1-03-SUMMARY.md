# Phase 5.1 Plan 03: Power Apps Generator (`__pwrapps__` aliases) + Hand-Authored OpenAPI Fragment Summary

**One-liner:** Ships the second per-product codegen pipeline for Phase 5.1 —
a hand-authored OpenAPI 3.0 fragment covering 19 flagship Power Apps
management ops, a `runPowerAppsPipeline` wrapper over plan 05.1-01's
shared `runProductPipeline` contract (exercising the `specUrl: null`
code path), and `__pwrapps__*`-prefixed aliases under permissive churn.

## Why It's Shaped This Way

**Why `specUrl: null` at the wrapper (not `undefined` or omitted):**
Plan 05.1-01's `runProductPipeline` treats both `null` and `undefined`
as the hand-authored-spec branch. The wrapper commits to the literal
`null` to make the contract explicit at the API boundary — mock
assertions (Test 13) can then pin the value, and grepping for `specUrl:
null` turns up every hand-authored-spec consumer in one scan. Using
`undefined` would leak through as "I forgot to set this" in other
contexts.

**Why 19 ops (not exactly 18 or 20):** The plan window was 15-25 ops
with a "target 18-20" recommendation. The plan's required list is 18;
adding `create-app` rounds out the CRUD for apps (list/get/create/delete)
and `delete-connection` matches `get-connection` symmetry for
connections. Both are natural companions to ops the plan already
required — no preset decision is forced now, and plan 5.1-07's
`pwrapps-essentials` preset still picks from within the full 19-op
surface.

**Why the `servers:` field points at `api.powerapps.com/providers/Microsoft.PowerApps`
even though runtime dispatch ignores it:** The research pinned this
base URL as the canonical Power Apps management root. Keeping it in the
spec serves two purposes: (1) documentation — operators reading the
YAML see the expected base without consulting the plan 5.1-06 audiences
table; (2) openapi-zod-client emits relative paths regardless of
`servers:`, so the codegen output is unchanged either way. Threat
T-5.1-03-d is mitigated because plan 5.1-06's dispatch reads
`PRODUCT_AUDIENCES.get('pwrapps').baseUrl`, NOT this field — a rogue
edit to `servers:` cannot redirect live traffic.

**Why fix the orchestrator tests in THIS plan rather than defer:** The
regression was directly caused by the pwrapps registration — tests that
run the real `runProductPipelines` iterator in a tmpdir without a
staged `openapi-pwrapps.yaml` throw `/hand-authored spec not found/i`.
Plan 05.1-02 set the precedent (it updated
`test/bin/generate-graph-client.test.mjs` Tests 2+3 in the same
landing); the fix is a single `runProductPipelines: async () => {}`
stub per affected test. Rule 1 (auto-fix bugs) applies.

## Integration Observations

### Power Apps op count

The hand-authored spec has 19 operations (under the 25-op ceiling;
matches the plan's 15-25 target and 18-20 recommendation with one op
of headroom for preset flexibility). The pipeline emitted **all 19**
`__pwrapps__*` aliases on the smoke-test integration run — no ops
dropped by the simplifier, no truncations required, no collisions
against the seeded main client.ts.

### No region-header at codegen

Test 11 asserts `!/x-ms-region/i.test(merged)` — and the smoke-test
integration run corroborated: `x-ms-region present in merged output:
false`. The platform-auto-route assumption (research A9) holds at the
codegen layer; the escalation path remains open for plan 5.1-06
dispatch work if wrong-region traffic is observed empirically.

### Orthogonal alias namespace

The `__pwrapps__` prefix is alpha-unique against every existing
namespace (v1 unprefixed, `__beta__`, `__powerbi__`). No collisions
possible by construction — the runProductPipeline collision guard
runs after prefix injection as a defense-in-depth check anyway.

### Spec shape

Hand-authored spec is **46 KB** (openapi/openapi-pwrapps.yaml); below
the 100 KB informal threshold for T-5.1-03-e (no DoS risk during
simplifier recursion). Every op takes a required `api-version` query
param; path params are typed `string`; POST request bodies are
`{type: object, additionalProperties: true}` — the loose shape
intentional because Power Apps app/environment manifests are deeply
nested JSON blobs that Zod would collapse to `z.record(z.any())`
regardless of strictness.
